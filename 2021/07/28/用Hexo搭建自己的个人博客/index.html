用Hexo搭建个人博客

发表于 2021-07-28 更新于 2021-07-29 分类于 个人博客

关于Hexo

Hexo是一个用Node.js写的博客框架，你可以用它来把自己写的Markdown文本生成一个静态网页（所以你可以把自己的博客放到代码托管平台上，比如Github和Gitee，这样就不需要自己去买服务器了）。它帮你完成了排版工作，你还可以通过安装其它主题和插件的方式来让自己的博客看起来更棒，拥有更多功能。

安装和使用Hexo

  1. 首先你的电脑需要安装node。
  2. 然后通过npm install hexo-cli -g来安装Hexo，然后新建一个目录或选择一个空目录来作为你希望Hexo工作的目录。
  3. 进入你选择的目录，通过hexo init来初始化你的Hexo目录。

注意：一定要在一个空目录里初始化Hexo，在你使用npm install hexo-cli -g时，会在当前目录里生成一个node_modules文件夹和其它文件，所以要重新选择一个空目录来进行初始化。
Hexo命令只能在这个初始化的目录里使用。
Linux和Mac用户需要在命令前加上sudo
4. 基本使用方法：

命令描述 命令
新建文章： hexo new “你的文章标题”
生成静态网页： hexo g
开启本地服务： hexo s
清除缓存： hexo clean

这样你就可以在本地玩Hexo了。

安装主题

Hexo默认使用的是landspace主题，我觉得它有些简陋，不过你喜欢这种风格的话也可以的。有很多人为Hexo开发了很多有个性的主题，你可以去这里找一下你自己喜欢的主题。我选择的是NexT主题，看起来挺舒服的。

    个性化配置

  4. 如果你用的是NexT主题，它其实有4种“Scheme”：
方案名字 具体效果
Muse 有大量留白，切换栏在上方
Mist 比较紧凑，切换栏在上方
Pisces 有侧栏，在主页面每篇博文连在一起

只需要在themes/next/_config.yml里去掉每个scheme前的#就可以了，你可以自己一个个试一下，找到自己最喜欢的。
2.

  • 你也许需要把自己的一些文章分门别类的放在一起的分类页面，和给自己的文章贴上标签的标签页，只需要在themes/next/_config.yml里找到m<!--More-->enu标签，把下面的categories（分类页）和tags（标签页）的前面的#去掉就可以了。那里面还有其它的页面，可以自己一个个试一下，看一看效果。
  • 然后在Hexo目录下的source目录下会生成categories和tags目录。两个目录下分别有一个index.md把里面的title: 后加上你希望在那个页面显示的标题，把type: 后面加上所属目录的目录名，即categories和tags。
  • 然后你每次生成新文章时，在生成的Markdown文件里会自动添加categories和tags，在后面加上你希望这篇文章所属的分类和带上的标签就可以了。
  1. 在Hexo目录下的*_config.yml*文件里，你可以自定站点信息。比如在Site下的language: 后你可以加上zh-CN让浏览器识别你的网页为简体中文网页。还可以在author: 后加上作者名字，在subtitle: `后面加上副标题什么的。可以自己慢慢配置。

    在Hexo框架里写Markdown的注意事项

  2. 如果你要插入图片，你可能会在source目录下丢一张图片，然后在Markdown文件里用![](图片名)来引用。我也是这么认为的，结果博客上加载不出图片，后来上网查了才知道解决方法。
  • 首先你可以在Hexo目录下执行npm install hexo-renderer-marked --save来安装默认的Markdown渲染器，然后在Hexo目录下找到post_asset_folder: false,把false改为true，这样再你每次生成新文章时，都会在source目录下生成一个和文章标题相同名字的目录，直接把这篇文章要用的图片放进去即可。然后直接在Markdown文件里引用，不需要加目录名，就当是把图片放在了文章的同级目录下。

  1. 如果你想让你的博客主页面只显示每篇文章的一部分，有一个点击阅读全文的按钮可以和点击文章标题一样触发显示整篇文章的行为，那么你可以在你编辑的Markdown文件中加入<!--More-->标签，来让这个标签前的内容是你想在主页面显示的内容，后面的内容不再主页面显示。（在当前的版本，你不可以通过插入两个<!--More-->标签的方式来在主页面只显示被两个标签包起来的文字。还有，如果你使用的是<!--more-->标签，那么那个点击阅读全文的按钮可能会比<!--More-->的要大。😂）
